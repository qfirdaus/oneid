Pemilik sistem menguji Sistem e-Prestasi (Pentadbir) melalui
`https://oneid.local/?site_id=DYYOWQGYLE`. Firefox menghentikan aliran kerana
redirect berulang. Pilot ini direkodkan sebagai **FAIL** dan menggantikan
pengesahan SSO umum sebelum cutover bagi tujuan acceptance hostname utama.

Bukti diagnosis tanpa merekod atau memaparkan token:

- access log menunjukkan kitaran berulang `GET /?site_id=DYYOWQGYLE` HTTP 302
  diikuti `POST /api.php` HTTP 200;

  `https://eprestasi-uat.upnm.edu.my/index.php`;

  ACL dan tidak diblacklist;
- saiz respons API sekitar 498–500 bait konsisten dengan respons token sah
  berserta user packet; respons site-denied semasa hanya sekitar 121–124 bait;
- medan identiti yang diperlukan consumer hadir dan melepasi format staf serta
  matrik;
- konfigurasi consumer yang diperiksa menggunakan site ID yang sama dan
  `https://oneid.local/` sebagai IDP.

Kesimpulan sementara: public-root OneID, token dan ACL berfungsi. Redirect loop
berlaku selepas handoff ke callback/login e-Prestasi. Log runtime pada host
consumer diperlukan untuk menentukan sama ada session/cookie handoff, pemetaan
akaun atau versi deployment consumer yang menolak login. Tiada perubahan dibuat
pada projek consumer dan rollback R4 tidak dicetuskan kerana tiada bukti bahawa
public-root menyebabkan kegagalan tersebut.
